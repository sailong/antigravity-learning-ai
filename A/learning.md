# 🚀 MCP Agent 全栈学习计划 (零基础版)

**目标项目**：智能简历筛选助手 (Resume Screener)
**核心技术**：Python, LLM (大模型), MCP (Model Context Protocol), Agent 逻辑
**先决条件**：已安装 Python 环境，拥有一个大模型 API Key (如 OpenAI, DeepSeek, Claude 等)

---

## 📅 第一阶段：基础概念与“大脑” (Day 1-2)
**目标**：理解 LLM 是如何通过代码进行交互的，而不只是在网页上聊天。

| 天数 | 课程主题 | 核心知识点 | 练习作业 |
| :--- | :--- | :--- | :--- |
| **Day 1** | **Python 与 LLM 的第一次握手** | 1. 什么是 API 调用？<br>2. 安装 `openai` 或 `anthropic` 库。<br>3. System Prompt (人设) vs User Prompt (指令)。 | 写一个 Python 脚本，让 AI 扮演“招聘经理”，无论你问什么，它都只用“面试官口吻”回答。 |
| **Day 2** | **结构化输出 (JSON)** | 1. 为什么 AI 喜欢废话？如何让它闭嘴。<br>2. 学习 JSON 格式（大括号、键值对）。<br>3. **重点**：Prompt Engineering，强制 AI 输出 JSON。 | 修改 Day 1 的脚本，让 AI 帮你把一段乱七八糟的自我介绍，整理成 `{ "name": "...", "skill": "..." }` 的 JSON 格式。 |

---

## 📅 第二阶段：不仅会说，还要会做 (Day 3-4)
**目标**：掌握 **Skills (技能)** 和 **Function Calling (函数调用)**，这是 Agent 的核心。

| 天数 | 课程主题 | 核心知识点 | 练习作业 |
| :--- | :--- | :--- | :--- |
| **Day 3** | **给 AI 装上“假肢” (Function Calling)** | 1. 什么是 Function Definition (工具描述)。<br>2. 模拟流程：用户提问 -> AI 返回函数名 -> 本地执行。 | 写一个伪代码 Agent：用户问“今天天气”，AI 返回 `{call: get_weather}`，你手动打印出“天气很好”。(理解原理即可) |
| **Day 4** | **MCP 协议入门 (Server 端)** | 1. 为什么要用 MCP？(标准化接口)。<br>2. 安装 `fastmcp`。<br>3. **实战**：编写 `resume_server.py`。 | 使用 `fastmcp` 写一个能读取本地文件夹（列出文件、读取内容）的简易服务器。并在终端测试它。 |

---

## 📅 第三阶段：构建智能体 (Day 5-6)
**目标**：编写 **Agent (Client 端)**，把“大脑”和“MCP 手臂”连接起来。

| 天数 | 课程主题 | 核心知识点 | 练习作业 |
| :--- | :--- | :--- | :--- |
| **Day 5** | **连接 MCP Client** | 1. 学习 `mcp` 客户端库。<br>2. **代码组装**：用 Python 代码启动并连接 Day 4 写好的 Server。<br>3. 验证 Client 能否看到 Server 的工具。 | 运行 `agent_client.py`，确保终端输出：“✅ 连接成功，发现工具: [read_resume]”。 |
| **Day 6** | **Agent 的思考循环 (Loop)** | 1. 实现 `Observe-Think-Act` 循环。<br>2. 处理上下文窗口 (Context Window) 问题。<br>3. **纠错机制**：处理 `JSONDecodeError` (我们在对话中讨论过的部分)。 | 完善 Agent 代码，让它能连续执行两步操作：先获取文件列表，再自动决定读取第一个文件。 |

---

## 📅 第四阶段：简历筛选项目实战 (Day 7-8)
**目标**：将前三阶段融合，完成最终的“简历筛选项目”。

| 天数 | 课程主题 | 核心知识点 | 练习作业 |
| :--- | :--- | :--- | :--- |
| **Day 7** | **项目逻辑实现** | 1. 准备 3-5 份测试用的 PDF/TXT 简历。<br>2. 编写最终的 System Prompt (评分标准)。<br>3. 整合所有代码。 | 运行项目：输入“帮我找个懂 Python 的人”，程序自动读取文件夹，分析每份简历。 |
| **Day 8** | **结果呈现与优化** | 1. 数据清洗：将 AI 的评分 JSON 收集起来。<br>2. 排序算法：在 Python 中按分数 `sort`。<br>3. 最终输出：生成一个 Markdown 格式的排名表。 | **最终成果展示**：程序运行后，生成 `ranking.md` 文件，里面按匹配度高低列出了候选人。 |

## 🎓 最终项目：简历筛选系统架构图

为了帮你理解这三者在项目中的配合，请看这张图：

1.  **用户 (You)**: "帮我筛选简历"
    ⬇️
2.  **Agent (大脑/Client)**:
    * 接收指令。
    * **Prompt**: "你是一个招聘专家，请调用工具读取文件..."
    * **LLM 交互**: 发送请求 -> 收到 JSON 指令 -> 解析 JSON。
    ⬇️
3.  **MCP Server (手/Skills)**:
    * `list_files()`: 扫描硬盘。
    * `read_file()`: 打开 PDF 读取文字。
    * **关键点**: 这里是唯一真正接触用户隐私数据的地方。
    ⬇️
4.  **结果处理**:
    * Agent 拿到文本 -> 再次问 LLM 打分 -> 修正 JSON 格式 -> 输出排名。
